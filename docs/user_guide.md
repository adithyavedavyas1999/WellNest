# User Guide

This guide covers how to use the three main WellNest interfaces: the Streamlit analytics dashboard, the public PWA, and the REST API. If you're a developer looking to integrate with the API, the `api_guide.md` has more detailed endpoint documentation.

## Dashboard (Streamlit)

The analytics dashboard is a multi-page Streamlit application designed for analysts, NGO staff, and policymakers. It runs at `http://localhost:8501` locally or on Streamlit Community Cloud for production.

### Getting Started

If you're running locally:

```bash
# make sure the API and database are running first
make docker-up
make run-api

# in a separate terminal
make run-dashboard
```

The dashboard connects directly to PostgreSQL for heavy queries (maps, aggregations) and to the FastAPI backend for AI features (RAG, briefs).

### Pages

#### 1. National Map

The landing page. Shows a choropleth map of US counties colored by average wellbeing score. You can:

- **Hover** over a county to see its name, score, and school count
- **Click** a county to drill into its schools
- Use the **state filter** dropdown to zoom into a specific state
- Toggle between **composite score** and individual **pillar scores** using the radio buttons
- Switch the **color scale** between the four category colors (Critical/At Risk/Moderate/Thriving) and a continuous gradient

The map uses Folium with the `streamlit-folium` integration. It can be slow to render on the first load because it's drawing ~3,200 county polygons. Subsequent interactions are faster because the base map is cached.

#### 2. School Explorer

The detail view. Search for any school by name, city, or state using the search box at the top.

For each school you'll see:
- **Score gauge** showing the composite wellbeing score with category label
- **Pillar breakdown** as a radar chart (education, health, environment, safety)
- **Key metrics** table showing the raw values that feed into each pillar
- **National and state ranking** with percentiles
- **Year-over-year change** indicator (arrow up/down with delta)
- **Risk flag** if the ML model predicts a significant decline
- **Location map** showing the school's position

You can toggle between schools using the search box without going back to the list.

#### 3. Resource Gaps

Shows schools prioritized for intervention. These are schools that have at least one pillar in the bottom national quartile while also having at least one pillar above the median -- the idea being that schools with uneven profiles are better intervention targets than schools that are uniformly struggling.

The table is sortable by:
- Gap count (how many pillars are in the bottom quartile)
- Pillar spread (gap between strongest and weakest pillar)
- Intervention priority (High Priority, Critical Need, Moderate Need)

You can filter by state and by which pillar has the gap (education, health, environment, safety).

#### 4. Trends

Year-over-year analysis. Shows:
- **Score distribution change** -- histogram comparing this year vs last year
- **Biggest movers** -- schools with the largest positive and negative score changes
- **Anomaly table** -- schools flagged by the anomaly detection pipeline, with narrative explanations
- **State-level trends** -- which states are improving or declining overall

The anomaly narratives are generated by the ML pipeline (template-based, not LLM). They explain why a school was flagged, e.g., "Math proficiency dropped 18 points YoY, while enrollment decreased 12%."

#### 5. AI Insights

Surfaces AI-generated content:
- **County briefs** -- GPT-4o-mini generated summaries suitable for grant proposals. Select a county from the dropdown to view its brief.
- **Anomaly narratives** -- LLM-enhanced explanations of flagged anomalies (when available)
- **Data quality alerts** -- any issues flagged by the LLM quality validator

The briefs are generated monthly. If you see "No brief available" for a county, it either hasn't been generated yet or the county was added after the last generation run.

#### 6. Ask WellNest

A RAG-powered Q&A interface. Type a natural language question about child wellbeing, education policy, or health guidelines, and get an answer grounded in federal policy documents (ESSA guidelines, Title I guidance, CDC school health guidelines, etc.).

Example questions:
- "What are the Title I funding requirements for schools with more than 40% poverty?"
- "What does ESSA say about chronic absenteeism interventions?"
- "What are CDC's recommendations for school-based mental health programs?"

The response includes the answer plus source citations (document name, page number, relevant excerpt).

This feature requires the OPENAI_API_KEY to be configured and the FAISS index to be built. If it's unavailable, you'll see a "RAG unavailable" message.

#### 7. Compare

Side-by-side comparison of two schools or two counties. Select the entities from dropdowns, and the page shows:
- Composite and pillar scores in parallel bar charts
- Key metrics comparison table
- Rank comparison (national and state)
- A map showing both locations

Useful for grant writers who need to contrast a target community with a reference community.

### Dashboard Tips

- **Caching:** The dashboard caches database queries for 5 minutes (configurable in `dashboard/utils/cache.py`). If you just ran the pipeline and don't see updated data, wait a few minutes or restart the dashboard.
- **Performance:** The National Map page is the heaviest. If it's sluggish, try filtering to a single state first.
- **Export:** Most tables and charts have a download button (top-right corner of each widget). Use this to export data for offline analysis.

## PWA (Public Web App)

The PWA is a lightweight, mobile-friendly school lookup tool. It's a static HTML/CSS/JS app that talks to the FastAPI backend.

### Accessing the PWA

- **Production:** Served from GitHub Pages (URL in the README)
- **Local:** Open `pwa/index.html` in a browser (requires the API to be running at `http://localhost:8000`)

### Features

1. **Search:** Type a school name, city, or state in the search bar. Results appear as cards below.
2. **School Detail:** Tap a card to see the full score breakdown, pillar radar chart, and key metrics.
3. **Map View:** Each school shows its location on a Leaflet map. You can also browse the map to find nearby schools.
4. **Offline Support:** Static assets are cached by the service worker. The search and detail features require an internet connection (they hit the API).
5. **Installable:** On mobile, use "Add to Home Screen" to install the PWA. It gets its own app icon and runs in a standalone window.

### Mobile Usage

The PWA was designed mobile-first. The layout stacks vertically on narrow screens, score gauges resize, and the map fills the available width. Touch targets are large enough for thumb navigation.

## API Quick Start

The REST API runs at `http://localhost:8000` locally. Full documentation is at `http://localhost:8000/docs` (OpenAPI/Swagger UI).

### Basic Usage

```bash
# health check (no auth needed)
curl http://localhost:8000/api/health

# list schools (paginated)
curl http://localhost:8000/api/schools?page=1&per_page=10

# get a specific school
curl http://localhost:8000/api/schools/170993000943

# search by name
curl "http://localhost:8000/api/search?q=Lincoln%20Elementary&state=IL"

# get county detail with AI brief
curl http://localhost:8000/api/counties/17031
```

If API key auth is enabled (production), add the header:

```bash
curl -H "X-API-Key: your-key-here" http://localhost:8000/api/schools
```

See `api_guide.md` for the full endpoint reference with request/response examples.

## Common Workflows

### Look Up a Specific School

**Dashboard:** Go to School Explorer (page 2), type the school name in the search box. The first result is usually the right one. Click it to see the full detail view.

**PWA:** Open the app, type the school name. Tap the matching card.

**API:** `GET /api/search?q=school+name&state=XX` then `GET /api/schools/{nces_id}` for the detail.

### Compare Two Counties for a Grant Proposal

1. Go to the **Compare** page (page 7) in the dashboard
2. Select "County" as the comparison type
3. Pick the two counties from the dropdowns (search by name or FIPS)
4. The side-by-side comparison shows composite scores, pillar breakdowns, and key metrics
5. Go to **AI Insights** (page 5) and look up the county briefs for both counties
6. Use the **Export** button to download the comparison data as CSV
7. Optionally, download PDF reports: `GET /api/reports/{fips}/pdf` for each county

### Find Schools That Need Intervention

1. Go to **Resource Gaps** (page 3)
2. Filter by your target state
3. Sort by "Intervention Priority" (High Priority first)
4. These are schools with at least one pillar in the bottom national quartile AND at least one pillar above median -- they have capacity to improve
5. Click any school to see which specific metrics are driving the gap
6. Check the **Trends** page (page 4) to see if the school is improving or declining

### Generate a County Report

```bash
# via API
curl -o cook_county_report.pdf http://localhost:8000/api/reports/17031/pdf

# or batch generate via the script
python -m scripts.generate_reports --state IL --output ./reports/output/
```

The PDF includes: composite and pillar scores, key metrics, the AI-generated community brief (if available), and data source attribution.

### Check for Data Quality Issues

1. Run the quality checks: `make dbt-test` for dbt tests, or trigger the quality_checks job in Dagster
2. Check the **AI Insights** page for any LLM-flagged anomalies
3. Look at the Soda checks output in `transformation/quality/`

### Ask a Policy Question

1. Go to **Ask WellNest** (page 6)
2. Type a natural language question about education policy, health guidelines, or funding requirements
3. The RAG pipeline searches federal policy documents and generates an answer with source citations
4. Use the source links to read the original documents for full context

## FAQ

**Q: How often is the data updated?**
A: The full pipeline runs weekly (Sunday 2 AM UTC). Weather alerts refresh daily. AI briefs regenerate monthly. Individual data sources update at different frequencies -- see the data dictionary for specifics. Most federal data sources lag 1-2 years behind the current date.

**Q: Why is a school missing a score?**
A: Usually because one or more required data sources don't have data for that school's county. The most common reason is missing ACS Census data for very small tracts (population suppression). If a school shows "Insufficient Data," it means we couldn't compute enough pillar scores to generate a meaningful composite.

**Q: Can I add my own data sources?**
A: Yes, but it requires code changes. See `contributing.md` for the process. You'd create a new connector in `ingestion/sources/`, a staging model in dbt, wire it into the appropriate silver model, and potentially adjust the scoring weights.

**Q: How are ties handled in rankings?**
A: We use SQL `RANK()` which assigns the same rank to tied schools. If two schools are both ranked #5, the next school is ranked #7 (not #6). This is standard competition ranking.

**Q: Why does my county show "No AI brief available"?**
A: AI briefs are generated monthly and cost about $16 per full run. If a county doesn't have a brief, it either hasn't been generated yet, or the generation failed for that county (usually because the underlying data was too sparse for a meaningful brief). You can trigger a regeneration via the Dagster UI.

**Q: Is the data available for download?**
A: Yes. The API supports pagination, so you can page through all schools or counties programmatically. The dashboard also has per-table download buttons. For bulk access, connect directly to the PostgreSQL database (read-only credentials available from the admin).

**Q: What's the difference between the dashboard and the PWA?**
A: The dashboard (Streamlit) is a full analytics platform for analysts -- multi-page, chart-heavy, supports complex filters and comparisons. The PWA is a lightweight mobile-friendly tool for the public -- search for a school, see its score. Think of the dashboard as the internal tool and the PWA as the public-facing surface.
